Confusion is your friend. Yeah, Really!

When you start programming your first web app, you're doing something that you've never done before.

It's a new challenge, a new opportunity, and in the case of software development, something so specific that it's probably never been done before.

Confusion, therefore, is a sign that you are heading in the right direction. It's a signal that you have engaged in the problem solving pursuit that is math and technology and that it's time for you to dig deeper and keep going because clarity awaits.


"Suspended disbelief" plays a more important role than you think

When learning a new technology, you will go through this horrible, uncomfortable phase where you KNOW the code is doing this THING, but you don't understand HOW exactly it's happening.

Whether learning how to write your first programming language, (or just the latest framework), this particular pain is something you need to become friends with. This is a normal but also a very critical part of the learning process.

When you encounter a new programming language, you need to re-train your eye. The way to do this is to read a good deal of code even when you don't totally understand what it's doing. Trust the process and one day confusion will make as much sense as your first natural language.


Going "Old Skool" when learning Tech

One thing I've learnt to do during my SC studies is to handwrite the instructions. First I write the instructions onto loose A4 paper, then I write it AGAIN into my A5 GIT notebook. THEN I type the instructions in my command line.

This method IS as slow, painful and tedious as it sounds, but it works. It's definitely helped me understand Git (and HTML, CSS) better. I have not, (by any means - yet) managed to memorise the sequence of Git commands (without referring to my notes) but I know that day will come, the more often I use Git.

There's something about handwriting that allows you to "see" the syntax better. Sure, you can re-read the code until your eyes bleed, but it's not the same. The action of writing makes your brain do something different when comprehending code and the step-by-step process of typing it.

So if you're struggling a little with your coding comprehension, why not give this method a go and see if it doesn't help or not. We may live in a world that is fast and full of copy/paste,but nothing beats understanding what it is, you're actually doing
